# Cullery MVP Spec

## 1. 文档目标

本文件用于定义 Cullery / 览羲 的最小可行产品范围，确保团队优先交付“可安装、可授权、可读取真实相册、可完成基础清理”的 Android 体验版。

## 2. MVP 核心目标

- 解决用户手机照片过多、清理效率低的问题。
- 让用户在不上传照片到云端的前提下完成基础整理。
- 提供真实可操作的相册访问、截图清理、重复候选识别和最佳照片推荐体验。
- 在 Android 虚拟机和真机上完成第一轮可用性验证。

## 3. MVP 范围

### 3.1 必做能力

- 相册权限申请与状态展示
- 麦克风权限申请与状态展示
- 真实相册读取
- 空间分析首页
- 截图识别与清理
- 完全重复项识别
- 相似照片候选分组
- 一组照片中推荐最佳保留图
- 应用内回收站
- 中英文切换
- 模型下载状态页
- Android 安装体验

### 3.2 可降级能力

- 语音控制可先支持有限命令集
- 专业选片可先采用规则 + 轻量评分模型
- 模型下载可先支持轻量模型，不强制首期接入 7B 模型

### 3.3 明确不做

- 云同步
- iOS 首发
- 桌面端正式版本
- 视频智能处理
- 故事集自动生成
- 完整 7B 端侧推理闭环

## 4. 目标用户

- 手机照片堆积严重的普通用户
- 需要快速清理截图和重复照片的用户
- 希望 AI 帮助选出最佳照片的摄影爱好者
- 对隐私敏感、不愿上传照片到云端的用户

## 5. 用户故事

- 作为用户，我希望应用读取我的相册并告诉我哪些内容最占空间。
- 作为用户，我希望一键筛出所有截图并批量清理。
- 作为用户，我希望从一组相似照片中自动保留最好的一张。
- 作为用户，我希望删除前有二次确认，避免误删。
- 作为用户，我希望通过语音说“删除所有截图”来完成基础操作。
- 作为用户，我希望在设置页看到权限状态和模型状态。

## 6. MVP 功能规格

### 6.1 空间分析

- 展示照片数量、截图数量、视频数量、已使用空间、可释放空间
- 展示重复项、截图、低质量候选的数量
- 提供“开始清理”入口

### 6.2 智能清理

- 截图识别
- 完全重复检测
- 相似照片候选分组
- 低质量照片初筛
- 批量勾选与删除建议
- 删除进入应用内回收站

### 6.3 最佳照片推荐

- 对 1 组相似照片进行评分
- 基于清晰度、曝光、构图、瞬间四类核心维度生成推荐
- 输出“推荐保留”与“建议删除”
- 展示简短推荐理由

### 6.4 语音控制

首期仅支持以下命令：

- 删除所有截图
- 保留这一组里最好的一张
- 打开设置
- 扫描最近照片

### 6.5 设置页

- 中英文切换
- 相册权限状态
- 麦克风权限状态
- 模型下载状态
- 模型下载按钮

## 7. 技术实现边界

- 客户端优先采用 Android + Capacitor 当前工程继续推进
- Web/PWA 继续作为演示与产品验证入口
- 相册读取需要替换当前 mock 数据
- 优先用轻量规则与小模型完成第一阶段
- 7B 模型只作为增强方向，不阻塞 MVP 验收

## 8. 验收标准

- 应用可在 Android 虚拟机安装运行
- 首次启动能成功弹出并处理相册权限
- 首页展示真实相册数据
- 至少支持一次真实截图清理流程
- 至少支持一次重复项候选识别
- 至少支持一次最佳照片推荐
- 设置页展示真实权限状态和模型状态
- 支持中英文切换

## 9. 风险与降级策略

- 若真实模型下载来不及，先保留状态页 + 轻量模型下载
- 若语音闭环来不及，先保证权限与识别展示可用
- 若相似聚类效果不稳定，先交付截图清理与完全重复检测

## 10. MVP 完成定义

当且仅当以下条件同时成立时，判定 MVP 完成：

- Android 端可安装运行
- 真实相册读取已打通
- 用户能完成一次真实清理
- 用户能看到一组候选照片的最佳推荐结果
- 关键路径无阻断性崩溃

# Cullery 开发任务拆解文档

## 1. 文档目的

本文件将 `project_spec.md` 与 `MVP_Spec.md` 中的能力拆解为可执行开发任务，便于排期、分工、开发与测试跟踪。

## 2. 任务拆解原则

- 先打通真实能力，再优化体验
- 先完成 Android 最小闭环，再扩展其他平台
- 先做规则与轻量模型，再接入重模型
- 所有关键任务都应具备对应验收结果

## 3. Epic 列表

| Epic ID | Epic 名称 | 优先级 | 目标 |
|---|---|---|---|
| E1 | 工程与运行环境 | P0 | 保证项目可构建、可安装、可调试 |
| E2 | 权限与原生能力 | P0 | 打通相册与麦克风权限 |
| E3 | 真实相册读取 | P0 | 替换 mock 数据，接入真实媒体库 |
| E4 | 空间分析与清理闭环 | P0 | 支持截图清理、重复项处理 |
| E5 | 专业选片 MVP | P1 | 推荐最佳保留图 |
| E6 | 语音控制 MVP | P1 | 支持有限命令执行 |
| E7 | 模型下载与状态管理 | P1 | 提供本地模型下载入口 |
| E8 | 国际化与设置 | P1 | 保证中英文与设置页完整性 |
| E9 | 测试与验收 | P0 | 完成虚拟机与真机验证 |

## 6. 本周建议冲刺

- Sprint Goal：让 App 在 Android 虚拟机中从“能展示”变成“能操作”
- Sprint 必做项：
  - E1-T1
  - E1-T2
  - E2-T1
  - E2-T2
  - E3-T1
  - E4-T1
  - E4-T2
  - E4-T4
  - E9-T1

## 7. 完成定义

当 P0 任务全部完成，且：

- 可安装运行
- 可读取真实相册
- 可清理截图或重复项
- 可恢复删除内容
- 可完成基础验收

则认为开发任务拆解第一阶段完成。

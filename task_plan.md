# Task Plan

## 目标

把当前 Cullery 从“可演示的 Web/PWA MVP”推进到“可安装、可授权、可读取真实相册、可执行基础操作的 Android 体验版”。

## Phase 1：真机可安装与原生权限打通

- 在本地 Android Studio 中完成 Gradle 依赖下载与工程首次运行。
- 在 Android 虚拟机中验证安装、启动、路由跳转、PWA/Capacitor 资源加载。
- 完成相册权限与麦克风权限的真实申请、拒绝处理与设置页状态回显。
- 补全 Android `AndroidManifest.xml` 所需权限说明与运行时行为验证。

## Phase 2：真实相册接入

- 引入可读取系统媒体库的原生方案，替换当前 `NativeService.getPhotos()` 空实现。
- 建立照片元数据读取能力：`id / path / size / width / height / createTime / location / mimeType`。
- 将真实相册数据接入 Zustand store，替换部分 mock 数据来源。
- 优先支持“最近照片”“截图”“重复候选”三类基础视图。

## Phase 3：可操作的清理闭环

- 完成照片勾选、保留、加入回收站、本地二次确认流程。
- 区分“建议删除”和“真正删除”，先做应用内回收站，再评估系统级删除能力。
- 建立最小可行规则引擎：截图识别、完全重复检测、低质量初筛。

## Phase 4：语音控制闭环

- 将语音识别结果接入指令解析器，而不是仅展示识别文本。
- 先支持有限命令集：
  - 删除所有截图
  - 保留这一组里最好的一张
  - 打开设置
  - 扫描最近照片
- 增加语音权限失败、识别超时、无结果时的兜底反馈。

## Phase 5：模型下载与离线 AI 能力

- 把当前模型下载模拟替换为真实下载任务。
- 设计模型存储目录、文件校验、失败重试、进度恢复和磁盘空间校验。
- 先接入轻量模型或规则+轻模型混合方案，避免一开始直接落地 7B 导致体验失控。
- 为后续 7B 模型预留：
  - 分片下载
  - 量化格式管理
  - 设备能力检测
  - 充电/Wi-Fi 条件下载

## 近期优先级

1. 本地 Android Studio 打开工程并跑通虚拟机。
2. 真实相册权限与媒体读取。
3. 截图清理与重复检测最小闭环。
4. 语音命令触发基础操作。
5. 轻量模型/规则引擎接入。

## 验收标准

- 应用可在 Android 虚拟机安装运行。
- 首次启动可正确申请相册与麦克风权限。
- 能读取真实照片列表，而不是只展示 mock 数据。
- 用户可以实际执行一次“清理截图”或“清理重复候选”的操作流程。
- 设置页能展示真实权限状态与模型下载状态。

## 本周冲刺建议

### Sprint Goal

让 Cullery 在 Android 虚拟机中从“能展示”升级到“能操作”。

### Sprint Backlog

- S1：在本地 Android Studio 跑通虚拟机安装。
- S2：完成真实相册权限申请与状态回显。
- S3：接入真实照片列表，替换首页/清理页 mock 数据。
- S4：实现“截图清理”最小可用闭环。
- S5：实现语音按钮触发真实识别或最小命令解析。

## 验收清单

| 编号 | 验收项 | 是否必须 | 判定方式 |
|---|---|---|---|
| AC-01 | Android 虚拟机可安装运行 | 必须 | 启动后进入首页 |
| AC-02 | 首次启动能正确申请权限 | 必须 | 系统权限弹窗出现并可回显 |
| AC-03 | 首页展示真实照片统计 | 必须 | 数据不再来自 mock |
| AC-04 | 清理页可执行至少一种真实清理动作 | 必须 | 完成一次截图清理流程 |
| AC-05 | 设置页展示真实模型状态 | 建议 | 展示未下载/下载中/已完成 |

## 备注

- Android 虚拟机测试是完全可行的，适合先验证安装、权限、页面跳转、媒体读取与基础交互。
- 真机测试仍然必须保留，因为虚拟机无法准确反映真实照片规模、存储空间和模型下载体验。
